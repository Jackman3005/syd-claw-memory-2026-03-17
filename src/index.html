The Problem

AI has amnesia — and it's not just an OpenClaw problem


Every session starts fresh

  • The model doesn't remember last week
  • You re-explain preferences every time
  • Context from old conversations is gone
  • The agent feels like a stranger again

What we actually want

  • Remembers what you told it last month
  • Learns your preferences over time
  • Gets better the longer you use it
  • Feels like a colleague, not a temp

This problem exists everywhere — ChatGPT, Claude, coding agents, OpenClaw. The context window is finite, sessions end, and memory is lost.

What Happens As You Chat

Context grows with every message — then what?

📨 Every message replays everything
System prompt + workspace files
Message 1 + Response 1
Message 2 + Response 2
Message 3 + Response 3
...
Message N + Response N
Your new message →
This is the same everywhere — ChatGPT, Claude, OpenClaw. The full conversation is sent every time.
When the context window fills up:
🗑️ Default: Truncate
Oldest messages get dropped. Context gone forever. This is what most tools do, including OpenClaw by default.
📝 OpenClaw: Compaction
Summarizes older messages before dropping them. Better — but the summary is lossy. Details are gone.
🌳 LCM: Lossless summarization
Summarizes into a DAG — but keeps every original message in a database. Agent can drill back into any summary to recover full detail. Nothing is ever lost.
⚠️ All of these are within one session. Start a new session and you're back to zero — unless you have cross-session memory.

What Exists Today

Current approaches and their limitations

💬 ChatGPT Memory
Auto-generates summary bullets from conversations. Injected into every session.
You don't control what it saves. Opaque. Can be wrong.
📁 Claude Projects
Upload docs + custom instructions. Loaded per-project.
Manual curation. Per-project, not per-you. Doesn't grow on its own.
🦞 OpenClaw Default
Workspace files (SOUL.md, MEMORY.md) + vector search over notes.
Better foundation, but memory growth is mostly manual.

The gap: None of these automatically observe, reflect, and consolidate memory over time.
They're either opaque (ChatGPT), manual (Claude), or a good starting point (OpenClaw).

What Would Ideal Memory Look Like?

Think about how humans actually remember things

👀
Observe
Notice what happens throughout the day. Most of it is noise — that's fine.
→
🔄
Reflect
Spot patterns, cluster related events, surface what might be important.
→
💤
Consolidate
During "sleep," decide what actually matters. Promote to long-term memory. Forget the rest.
→
🧠
Remember
Retrieved when relevant. Changes future behavior. Gets better over time.

This is basically how human memory works — and it's the model we're trying to build for AI agents.

Core Principles

Guiding principles for building agent memory

Default to forgetting
Require justification for remembering. Miss some signal > ingest endless noise.
Observations are not memory
Observations = raw sensor residue. Memory = processed understanding that changes future behavior.
Cheap capture, expensive consolidation
Fast/cheap models handle observation. Strong/expensive models handle judgment and promotion.
Structural pre-filter before model
Regex kills cron exhaust. No need to spend tokens on obvious noise.
Contradictions are first-class records
When facts change, log the contradiction. It's belief revision, not a bug.
Progressively opinionated pipeline
Observer is permissive. Reflector is interpretive. REM is judgmental. Each layer adds conviction.

🔄

Spots patterns.

Still non-canonical.
→
💤

Nightly (2am)

"This happened" → Observer → "This is a pattern" → Reflector → "This matters" → REM

How Memories Get Into Context

Two mechanisms — automatic injection and on-demand retrieval

📂 Loaded automatically every session
Injected into the system prompt before your first message:
SOUL.md — personality
AGENTS.md — instructions
USER.md — who you are
MEMORY.md — long-term memory
TOOLS.md, IDENTITY.md, HEARTBEAT.md
Always present. Costs tokens every turn. Keep these concise.
🔍 Retrieved on demand (tool calls)
The model decides to search when it thinks it needs more context:
memory_search — semantic vector search over all memory files
memory_get — read specific file/line range
The system prompt tells the model:
"Before answering questions about prior work, decisions, dates, people, preferences, or todos — run memory_search first."
Only loaded when needed. Doesn't cost tokens until the model calls it.

Key insight: daily logs, topic files, and channel context are never auto-loaded — they're pulled in via semantic search when relevant. This keeps baseline token cost low while still having deep recall available.

Where Memory Lives

Plain Markdown files — no database, no magic

memory/
├── daily/ raw daily logs
├── channels/ per-channel context
├── topics/ cross-cutting knowledge
├── rem-logs/ consolidation audit trail
└── MEMORY.md distilled long-term
Daily logs
Raw chronology. Session residue. No elegance required. Ages out.
Channel context
Per-channel memory. What's normal in #home-lab vs #coding. Tiered: daily → weekly → long-term.
Topic files
Cross-cutting project and subject knowledge. "What do I know about the Plex stack?"
MEMORY.md
Distilled canonical memory. Only things that change future behavior. Curated by the REM cycle.

Everything is Markdown. Readable, editable, git-tracked, portable. No vendor lock-in.

The Data Flow

From raw event to lasting memory

Event occurs
→
Pre-filter (no model) — Structural kill list (regex), human involvement check, cooldown, novelty check. Most events die here.
Survivors
→
Observer — "What happened?" Events only, no conclusions. → inbox/observations/
Patterns
→
Reflector — "What's recurring?" Cluster, dedupe, suggest routing. Still non-canonical.
Judgment
→
REM Cycle — "What changes behavior?" Promote, prune, flag contradictions, audit trail.
Long-term
→
MEMORY.md · topics/ · channels/long-term/ — Retrieved via vector search when relevant. Changes future behavior.

Challenges & Lessons

Things I've learned building this

🗑️ 95% of observations are noise
Cron logs, heartbeat acks, repeated tool calls. The pre-filter is critical — without it, you're paying tokens to process garbage.
🤯 Memory ≠ logging
Capturing everything is easy. Deciding what matters is the hard part. The REM cycle is where the real value is — and the hardest to get right.
✅ Plain Markdown is underrated
No database, no schema, no migrations. The agent can read and write its own memory with basic file tools. Git gives you versioning for free.
✅ Iterative design with the agent worked well
Designing the memory system collaboratively with my own agent — debating tradeoffs, testing ideas — produced a much better result than solo spec-writing.
⚠️ Contradictions are harder than they sound
People change preferences. Facts evolve. "I like Thai food" and "I've gone off Thai food" aren't a bug — they're belief revision. The system needs to handle that.
⚠️ Token costs are real
Observer + Reflector on cheap models is fine. The nightly REM cycle on Opus is where the bill lives. Worth it — but budget accordingly.

The Bigger Picture

Your AI memory is yours — not the platform's


OpenClaw won't be the last platform. In 5 years, we'll all be using something we haven't heard of yet.

But the context you build now — your preferences, your communication style, what your agent knows about you — that should be portable.

Why Markdown matters
It's plain text. No vendor lock-in. No proprietary format. When the next platform comes along, you copy your memory files over and your new agent picks up where the old one left off.

The memory you're building today is an investment in your future AI experience — regardless of which platform you end up on.

What's Next

This is exploration, not a finished product

🔧 Better pre-filtering
The observer noise problem is the #1 issue. Structural filters need to kill 95% of events before any model sees them.
💻 Beyond OpenClaw
This isn't just an OpenClaw problem. Claude Code, ChatGPT, every AI tool needs better memory. Can we bring these patterns to other platforms?
🧪 More testing & iteration
This has been running for a few weeks. Longer-term patterns, better contradiction handling, and tuning the REM cycle are all in progress.
🤝 Community approaches
Other people are solving this differently — Letta, LangMem, Mem0. No one's cracked it yet. Lots to learn from each other.

I'm not claiming this is the solution — it's one approach, it's working for me, and I want to keep improving it.
